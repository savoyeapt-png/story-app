앱 사용설명서
USER GUIDE — 기능 설명, 추천 흐름, 빠른 사용 팁
이 앱은 작품 설계 → 서사 구조 정리 → 원고 집필 → 분석 요청 → 피드백 반영 흐름으로 쓰면 가장 안정적으로 돌아갑니다. 저장은 기본적으로 브라우저 내부에 이루어지므로, 중요한 작업물은 상단의 프로젝트 JSON 내보내기로 백업해 두는 것을 권장합니다.
1. 작품 설계
작품 개요: 로그라인, 장르, 핵심 정서를 빠르게 고정합니다.
세계관 · 규칙: 배경 설정, 금기, 룰, 시간대 같은 기초 규칙을 정리합니다.
캐릭터: 인물 정보와 이미지 파일을 함께 붙여 캐릭터 시트를 만듭니다.
관계 · 변화: 인물 노드와 연결선, 텍스트 박스를 사용해 관계도를 시각적으로 정리합니다.
2. 서사 구조
비트시트 캔버스: 큰 줄기의 감정선과 전환점을 잡습니다.
사건 캔버스: 장면 단위 사건을 메모처럼 흩뿌리듯 배치합니다.
사건 타임라인: 사건들을 순서대로 정리해 흐름을 점검합니다.
에피소드 조각: 회차별 아이디어 조각, 대사, 이미지 포인트를 저장합니다.
복선 · 회수: 떡밥을 심고 회수되는 흐름을 추적합니다.
3. 원고 작업
원고 · 회차: 본문 초안과 프리뷰 탭을 같이 보며 회차별 원고를 관리합니다.
쓰기 모드: 드래프트를 집중해서 쓰고, 자동 정리와 에피소드 카드 생성까지 이어갑니다.
원고 프리뷰에는 기본 회차 / 로맨스 / 갈등 / 클리프행어 고정 탭이 있어 참고용 템플릿으로 활용할 수 있습니다.
4. 분석 요청
분석 요청서: 프롬프트 프리셋을 고르고, 분석 대상 회차와 추가 요청을 정리합니다.
패키지 생성: 현재 화, 참고 화, 카드, 추가 자료를 묶어 내보냅니다.
앱 안에 원고가 있으면 자동으로 패키지에 포함되고, 없으면 별도 업로드로 첨부할 수 있습니다.
5. 추천 사용 순서
작품 개요와 캐릭터를 먼저 정리합니다.
비트시트와 사건 캔버스로 큰 줄기를 잡습니다.
에피소드 조각과 원고 회차에서 구체 장면을 만듭니다.
쓰기 모드에서 초안을 작성하고, 자동 정리와 카드 생성을 활용합니다.
분석 요청서에서 프롬프트와 범위를 설정해 패키지를 내보냅니다.
받은 피드백은 다시 쓰기 모드에 반영해 리라이팅합니다.
6. 자주 쓰는 팁
백업: 작업 기기를 바꾸거나 큰 수정 전에 JSON 내보내기를 권장합니다.
분석 범위: 현재 화만 보내기보다, 이전 2~3화를 참고 화로 함께 보내는 것이 개연성 점검에 유리합니다.
관계도: 색상은 감정 성격, 선 종류는 관계 상태, 텍스트는 사건 변화를 적는 식으로 구분하면 읽기 쉽습니다.
쓰기 템플릿: 프리뷰 고정 탭을 옆에 켜두고 집필칸은 자유롭게 쓰는 방식이 가장 덜 답답합니다.
